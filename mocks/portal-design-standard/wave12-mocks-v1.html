One emoji per idea, everywhere. Wave 0 writes a shared dictionary into the design guide so the same concept always uses the same emoji across all modules — no module invents its own.
🏭 Production📦 Inventory🛒 Purchase👨‍🍳 Chef Guide 🥫 Sauces/prep📋 Lists🔍 Count📥 Receive ✅ Tasks/approve🔄 Handover📅 Planning📊 Reports 🏷️ Labels📖 Recipes⚙️ Settings

Interface machinery (home, back, close, chevrons, checkmarks) uses one thin-line icon set — the white outline icons already in the header. Never emoji, never a second style.

This is what keeps 15 modules feeling like one app, not fifteen.

What happens next, if you approve these: Wave 0 (the shared building blocks + rulebook) lands first with no visible change, then Wave 1 ships the two home screens, then Wave 2 ships these five — each as its own update on staging, each real-browser tested before it's called done. Anything here you'd tweak — an emoji, a stat choice, a label — is easiest to change now, on the mock.
